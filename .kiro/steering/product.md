# SoFit 서비스 개요

## 서비스 소개

SoFit은 소상공인을 위한 대출 플랫폼입니다.
기존 신용평가 체계에서 소외된 소상공인에게, 성장 가능성을 평가한 ML 기반 성장 S등급(S1~S10)을 활용해 대출 접근성을 높입니다.

## 사용자 역할

- **USER**: 소상공인 고객. 대출 신청, 성장 S등급 조회, My Biz Data 연동
- **BANK_ADMIN**: 은행원. 대출 심사 처리, 상품 관리
- **DEV_ADMIN**: 개발자. 사용자 관리, API 로그 조회

## 핵심 도메인 개념

- **성장S등급**: LGBM ML 모델로 산출하는 소상공인 신용등급 (S1~S10). Spring Batch(일일/월별)로 배치 실행
- **SCB 점수**: CB 점수에 성장S등급 가산점이 붙은 점수. 성장S등급과 반드시 구분
- **My Biz Data**: 소상공인 사업자 데이터 (매출, 현금흐름, 업종 순위 등). 성장S등급 산출의 입력 데이터. Mock으로 생성하여 DB에 직접 저장. 최초 1회만 수집 (갱신 추후 구현)
- **SHAP 설명**: 성장S등급 산출 근거. SHAP 결과 → LLM(Gemini) → 자연어 설명 생성 순서로 처리. SHAP 결과와 LLM 결과 모두 DB에 저장. 고객에게는 LLM이 생성한 자연어 설명 제공. 은행원은 대출 심사 화면에서만 성장S등급 산출 근거 확인 가능
- **External Mock**: 국세청, CB사, 금융인증서 API를 별도 노트북에서 Mock 운영

## Spring Batch 구성

- **일일 배치**: 신규 가입 후 성장S등급 미산출 USER + 대출 신청한 USER 대상으로 성장 S등급 산출
- **월별 배치**: 기존 전체 회원 성장S등급 갱신
- 사용자 화면에서는 배치로 산출된 결과를 조회 (실시간 산출 아님)

## 주요 플로우

### 1. 회원가입

```
기본 정보 입력 (이름, 주민번호 앞 7자리, 연락처)
→ KYC 인증 (사업자등록번호 진위 확인 - ext_kyc_records Mock)
→ 금융인증서 불러오기 (ext_financial_certs Mock, VALID + 실명 일치 확인)
→ PIN 인증 (ext_financial_certs.pin_hash Mock 검증)
→ 아이디/비밀번호 설정
→ 약관 동의 (필수 약관 미동의 시 진행 불가)
→ 가입 완료
```

### 2. My Biz Data 수집

```
진입 화면 (서비스 소개)
→ PIN 인증 (금융인증서 PIN)
→ 약관 동의 (마이 비즈니스 데이터 서비스 약관)
→ 데이터 수집 중 (Mock 데이터 로딩, 항목별 진행 상태 표시)
→ 통합 대시보드 (월 매출, 순이익, 현금흐름, 업종 순위 등)
```

- Mock 데이터를 외부 API 없이 DB에 직접 저장
- 최초 1회만 수집 가능 (갱신 기능 추후 구현)

### 3. 대출 신청

```
대출 목록 → 상품 상세 → 대출 신청 버튼
→ 사전 입력 (기본 정보 확인)
→ 대출 약관 동의
→ PIN 인증 (금융인증서 PIN)
→ 사업자 정보 확인
→ 마이데이터 정보 동의 (약관 동의)
→ 마이데이터 불러오기 (My Biz Data 연동, 로딩)
→ 희망 대출 조건 입력 (금액, 기간, 용도, 상환 방식)
→ 대출 신청 완료 (loan_applications SUBMITTED 상태)
```

- KYC 미완료 시 접근 차단
- My Biz Data 미존재 시 신청 불가
- 동일 상품 중복 신청 불가

### 4. 심사 결과 조회 및 약정 실행 (USER)

```
대출 진행 현황 목록
→ 심사 진행 중 (스텝 UI로 단계별 상태 표시)

[승인 시]
→ 대출 승인 화면 (확정 금리 제안, 한도 직접 입력)
→ 약정 체결
→ 약관 동의
→ PIN 인증 (금융인증서 PIN, 전자서명)
→ 계좌 인증 (1원 송금 요청 → 인증번호 확인)
→ 대출 실행 완료

[거절 시]
→ 거절 사유
```

### 5. 심사 처리 (BANK_ADMIN)

```
대출 현황 대시보드 (목록, 페이징, 상태·날짜·사업자명 필터)
→ 신청 상세 (신청자 정보, My Biz Data, 성장 S등급, SHAP 내부 변수 전체 포함)
→ [승인] 금리 제안 + 한도 직접 입력 → 승인 처리 (APPROVED)
→ [거절] 거절 사유 입력 → 거절 처리 (REJECTED)
```

- 이미 결정된 건 재처리 불가
- 추가 서류 요청 기능 없음

### 6. 성장 S등급 분석 리포트

```
성장 S등급 분석 리포트 진입
→ [My Biz Data 미연결] 연결 유도 화면 → My Biz Data 수집 플로우로 이동
→ [My Biz Data 연결됨]
   → [배치 미실행 / 미산출] "아직 성장 S등급이 산출되지 않았어요" 안내 표시
   → [배치 완료] DB에서 최신 성장 S등급 조회 → 성장 S등급 분석 결과 (S1~S10 등급, 등급 수준 표시)
   → 상세 리포트 (LLM 자연어 설명 — 항목별 기여도, 긍정/부정 방향, 개선 조언)
```

- 성장 S등급: ML(LGBM)로 산출. 성장 S등급 분석 리포트에서 고객에게 제공
- SCB 점수: CB 점수 + 성장 S 등급 가산점. 대출 심사 화면에서만 노출 (고객 화면 없음)
- 고객 상세 리포트: LLM이 생성한 자연어 설명(XAI → LLM) 제공
- 은행원 성장 S등급 산출 근거: 대출 심사 화면에서만(XAI) 확인 가능

### 7. 마이페이지

```
마이페이지 메인 (프로필, 메뉴 목록, 푸시 알림 설정 토글)
→ 내 정보 확인 (이름, 아이디, 연락처)
→ 사업자 정보 확인 (사업자번호, 상호명, 업종, 개업일, 대표자명)
→ 알림 목록 (심사 상태 변경 알림 등)
→ 로그아웃 (확인 다이얼로그 → 서버 세션 삭제)
→ 회원 탈퇴 (유의사항 확인 → users.status = INACTIVE)
```

## 레포지토리 구성

- `SoFit-backend`: Spring Boot 멀티모듈 (user-backend, admin-backend, common)
- `SoFit-frontend`: React 모노레포 (user, admin)
- `SoFit-AI`: FastAPI + LGBM 모델
- `SoFit-DevOps`: Docker, Docker Compose, Redis 설정
- `SoFit-external-mock`: 외부 API Mock 서버 (별도 노트북, CI/CD 없음)
